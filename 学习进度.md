### 剖析《车辆路径问题的模型及算法研究综述》综述文章的架构

VRP问题决策内容通常包括两层：
第一层：分配-----》哪些客户由哪辆车服务
第二层：路径-----》每辆车以什么顺序访问客户

#### VRP与 TSP 的关系


| 维度   | TSP     | VRP           |
| ---- | ------- | ------------- |
| 车辆数  | 1       | ≥ 1           |
| 容量   | 无       | 常有（CVRP）      |
| 出发点  | 一个环     | 多车从车场出发       |
| 是否回场 | 回到起点    | 通常要回（也有开路径型）  |
| 复杂度  | NP-hard | NP-hard，且规模更大 |




#### VRP 分类体系

综述按多个维度对 VRP 分类，这是读后续文献的“地图”。

1. 按任务特征

装货问题（从车场装货送出去）
卸货问题（从客户处收集货物）
装卸混合问题（Backhauls，去程送货、回程取货）
2. 按服务对象
对弧服务：如中国邮递员问题（CPP）
对点服务：如 TSP
混合服务：如校车路线
3. 按车辆与车场
满载 / 非满载
单车场 / 多车场
单车型 / 多车型（MFVRP / HFVRP）
车辆封闭（必须回车场）/ 车辆开放（可不回）
4. 按信息是否确定
确定性 VRP：需求、位置已知
不确定性 VRP：
SVRP（随机需求）
FVRP（模糊需求）
5. 按约束类型（最常用，建议背下来）

### VRP 的数学模型（三种基本构造）

 根据《车辆路径问题的模型及算法研究综述》大多数模型可看成下面三类的变形与组合

## 4.1 车流模型（Vehicle Flow）

- **决策变量**：车辆 k 是否走弧 (i, j)
- **代表形式**：三下标、二下标车流方程
- **特点**：适合精确算法（分枝定界、列生成）
- **三下标**： (i, j, k) —— 车 k 从 i 到 j
- **二下标**：去掉车序号，更紧凑，多用于对称 CVRP

---



## 4.2 物流模型（Commodity Flow）

- **决策变量**：货物如何在网络中流动
- **适用**：多种货物、多种流向

---



## 4.3 集合覆盖 / 分割模型（Set Partitioning / Covering）

- **思路**：先枚举所有可行路径，再选一组路线覆盖全部客户
- **优点**：模型直观
- **缺点**：可行路径数量指数增长，需要列生成

| 类别   | 精确算法       | 启发式算法   |
| ---- | ---------- | ------- |
| 目标   | 证明最优       | 快速得到满意解 |

### 文章观点

实际中更依赖启发式方法，因为精确算法存在**指数爆炸问题**。

### 重点关注启发式算法

6.1 先分组，后路线（Cluster First, Route Second）
代表：Sweep 算法（Gillett & Miller, 1974）

步骤：

以车场为原点，算各客户极角
按角度排序，用射线“扫”过平面
在容量约束下把客户聚成若干组
每组内再排 TSP 路线
适用：装货/卸货、CVRP

```
    客户分布图（俯视）
          ·  C3
     · C2    · C4
[车场] · C1      · C5
     · C6    · C7
```

按极角扫过 → 分组 → 每组一条路
6.2 先路线，后分组（Route First, Cluster Second）
步骤：

把所有客户当作一个 TSP，求一条“大环”
按车辆容量把大环切成多条子路线
优点：全局连通性好
缺点：切割方式影响最终质量

6.3 节约/插入算法（Savings / Insertion）
代表：Clarke-Wright 节约算法（1964）=

核心公式（合并路径 i→1 和 1→j）：

$$
s_{ij}=c_{i1}+c_{1j}-c_{ij}
$$

含义：若让客户 i 和 j 在同一条路上连续访问，比分别回车场能节约多少。

步骤：

初始：每个客户单独一条路线（i→车场→i）
计算所有 
sij，sij ，从大到小排序
依次尝试合并，检查容量等可行性
直到无法合并
复杂度：约 
O(n2logn)
O(n2logn)
适用：CVRP，车辆数不固定时很常用

6.4 改进/交换法（Local Search）
在可行解上反复做：

2-opt、3-opt（交换边）
点在不同路线间移动
直到目标无法改进
常与构造算法（CW、Sweep）串联使用。

6.5 基于数学规划的启发式
建 MIP → 拉格朗日松弛 → 列生成 → 分枝定界
介于精确与纯启发式之间
6.6 交互式优化
把调度员经验融入参数和约束调整（偏工程应用）
